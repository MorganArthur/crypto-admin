# Crypto Admin - Startup Scripts Guide

## 📁 Script Files

The project provides three convenient Windows batch scripts to manage the application:

### 1. `start.bat` - Quick Start
**Use case**: All dependencies are already installed, quick service startup

**Features**:
- ✓ Check Python and Node.js environment
- ✓ Start backend FastAPI service (port 8000)
- ✓ Start frontend Vite development server (port 5173)

**Usage**:
```bash
# Double-click to run or execute in command line
start.bat
```

---

### 2. `start-full.bat` - Full Startup (Recommended for first-time use)
**Use case**: First run or need automatic dependency installation

**Features**:
- ✓ Check and install Python virtual environment
- ✓ Automatically install backend dependencies (requirements.txt)
- ✓ Automatically install frontend dependencies (npm install)
- ✓ Create default .env configuration file
- ✓ Start backend and frontend services

**Usage**:
```bash
# Double-click to run or execute in command line
start-full.bat
```

**Note**: First run may take a long time (5-10 minutes) to install dependencies

---

### 3. `stop.bat` - Stop All Services
**Use case**: Quickly stop backend and frontend services

**Features**:
- ✓ Stop backend service process
- ✓ Stop frontend service process

**Usage**:
```bash
# Double-click to run or execute in command line
stop.bat
```

---

## 🚀 Quick Start

### First Time Use
1. Make sure Python 3.8+ and Node.js are installed
2. Double-click to run `start-full.bat`
3. Wait for dependency installation to complete
4. Visit http://localhost:8000/docs (Backend API docs)
5. Visit http://localhost:5173 (Frontend UI)

### Daily Use
- **Start**: Double-click `start.bat`
- **Stop**: Double-click `stop.bat`

---

## 🔧 Manual Startup Method

If you prefer not to use scripts, you can also start manually:

### Backend
```bash
cd backend
python api_server.py
# or use uvicorn
uvicorn api_server:app --reload --port 8000
```

### Frontend
```bash
cd frontend
npm run dev
```

---

## 📝 Notes

1. **Port Usage**: 
   - Backend uses port 8000
   - Frontend uses port 5173 (will automatically select another port if occupied)

2. **Virtual Environment**: 
   - `start-full.bat` will automatically create and use `venv` virtual environment
   - It's recommended to always use virtual environment to avoid dependency conflicts

3. **Environment Variables**: 
   - To use DeepSeek API, edit the `backend\.env` file
   - Copy template from `backend\.env.example` and fill in your API Key

4. **Firewall**: 
   - Windows Firewall may ask whether to allow Python/Node.js network access on first run
   - Please choose "Allow" to ensure services run properly

5. **Stopping Services**: 
   - You can directly close the opened command windows
   - Or use `stop.bat` to stop all uniformly

## 🐛 Common Issues

### Q: Prompt "Python not detected"
A: Make sure Python 3.8+ is installed and added to system PATH

### Q: Prompt "Node.js not detected"
A: Make sure Node.js is installed and added to system PATH

### Q: Frontend dependency installation failed
A: Try deleting `frontend\node_modules` folder and `frontend\package-lock.json`, then re-run `start-full.bat`

### Q: Backend dependency installation failed
A: Check network connection, or manually activate virtual environment and run `pip install -r backend\requirements.txt`

### Q: Port already in use
A: Close the program occupying the port, or modify the port number in the code

## 📞 Technical Support

If you encounter problems, please check:
1. Whether Python and Node.js versions meet requirements
2. Whether network connection is normal (for downloading dependencies)
3. Whether there is sufficient disk space
4. Whether antivirus software is blocking script execution
